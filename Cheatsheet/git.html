Git cheatsheet

Git Fundamentals

Three locations: Working directory, index, repository.
Git commands copy files between these locations

the repository is a sort of online directory when you are working on a project with multiple people, you can put your local files to the repository (thanks to the "commit" command and the "push" command) then, everyone can get your files or your modifications (by using the "pull" command).
The Working Directory is the specific folder where you're working on your own pc. The goal is to share it on the REPOSITORY (online plateform for sharing project)

The process

  • Open CMD
  • Go to the directory that you want to share by command "cd 'name of dir'..."
  • We have to tell Git where the "root" of the repository is, for that type "git init":
    A hidden folder will appear on the working directory :


  • Then type "git status" as you can see no commit has been commited yet:
    The red text is the file who aren't prepared to be commited:


  • For copying the file from the directory to the queue for the commit type "git add name of the file.ext". Nothing change, it's normal.
  • Type again "git status" the file is green, now we can commit the directory


  • To commit the file from the working directory to the local repository "git commit -m "your comment":Nothing happen, it's normal. The file is now commited.

Git push

Before the pushing to the online repository, we have to configure the local repository

  • After creating a acount on github, return to CMD and type: git remote add origin https://github.com/user/poco-yourname.git (with your own id)
  • Optional: Type git remote -v to verify all your configurations already done in the same directory.
